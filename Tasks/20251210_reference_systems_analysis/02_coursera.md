## 1. Краткий обзор

**Тип системы:** MOOC-платформа (Massive Open Online Courses) с корпоративным решением Coursera for Business

**Целевая аудитория:**
- **B2C:** Индивидуальные учащиеся по всему миру (170+ млн пользователей)
- **B2B/B2G:** Более 15 000 организаций, включая корпорации и государственные структуры
- **Академический сектор:** Университеты и образовательные учреждения

**Ключевые отличия от конкурентов:**
- Глобальная платформа с курсами от ведущих университетов (Stanford, Yale, Google, IBM, AWS и др.)
- Каталог 10 600+ курсов, 1 400+ специализаций, 165+ профессиональных сертификатов
- Мультиязычность: контент на 54+ языках, 4000+ топ-курсов доступны на 20 неанглийских языках
- AI-первый подход: AI-ассистированные учебные пути, автоматическая генерация контента
- Эталонная система прогресс-трекинга и функция "следующий шаг"

**Почему в референсах:**
Coursera выбрана как **эталон UX для массовой аудитории** и **лучшие практики прогресс-трекинга**. По данным Coursera, введение функции "next step" (следующий шаг) повысило процент завершения курсов более чем на 10%. Платформа рассчитана на миллионы пользователей с разным уровнем технической грамотности, что делает ее UI/UX особенно ценным для B3 LMS. Корпоративное решение Coursera for Business показывает, как масштабировать обучение для предприятий и государственных организаций.

## 2. Структура системы

### 2.1. Основные роли

| Роль | Описание | Ключевые функции |
|------|----------|------------------|
| **Learner (Учащийся)** | Конечный пользователь, проходящий курсы | Просмотр материалов, выполнение заданий, получение сертификатов |
| **Instructor (Преподаватель)** | Создатель и ведущий курса | Создание контента, проверка заданий, ответы в форумах |
| **Peer Reviewer (Peer-эксперт)** | Учащийся, проверяющий задания других студентов | Оценка работ по рубрикам (для peer-review заданий) |
| **Admin (Администратор)** | Управление корпоративной программой обучения | Назначение курсов, отслеживание прогресса, отчетность |
| **Program Manager** | Менеджер учебных программ (Coursera for Business) | Создание учебных путей, управление лицензиями |

### 2.2. Модель данных (сущности)

#### Основные сущности:

**1. User (Пользователь)**
- Профиль с данными: имя, email, фото, биография
- История обучения: пройденные курсы, прогресс, сертификаты
- Настройки: язык интерфейса, уведомления, таймзона
- Связи: Course Enrollments, Certificates, Forum Posts

**2. Course (Курс)**
- Метаданные: название, описание, университет-партнер, инструкторы
- Структура: модули (Modules) → уроки (Lessons) → единицы контента (Items)
- Продолжительность: рекомендованные недели, часы в неделю
- Настройки: язык, субтитры, дедлайны (гибкие или фиксированные)
- Связи: Specialization, Assessments, Enrollments

**3. Specialization (Специализация)**
- Набор из 4-10 курсов по одной теме
- Последовательность: курсы с пререквизитами (1→2→3→Capstone)
- Capstone Project: финальный проект для завершения специализации
- Сертификация: единый сертификат по завершении всех курсов
- Связи: Courses, Specialization Enrollment

**4. Professional Certificate (Профессиональный сертификат)**
- Аналог Specialization, но от корпоративных партнеров (Google, IBM, Meta)
- Ориентирован на карьерные навыки
- Включает практические проекты

**5. Enrollment (Запись на курс/специализацию)**
- User → Course/Specialization
- Дата начала, прогресс (%), статус (active/completed/dropped)
- Тип доступа: бесплатный аудит, платный сертификат, подписка Coursera Plus
- Связи: User, Course, Progress Data

**6. Item (Единица контента)**
- Типы: Video, Reading, Quiz, Graded Quiz, Peer-Graded Assignment, Discussion Prompt
- Статус завершения: not started / in progress / completed
- Для видео: прогресс просмотра, заметки, закладки
- Связи: Course Module, User Progress

**7. Assessment (Оценочное задание)**
- Типы:
  - **Auto-graded Quiz:** мгновенная проверка (множественный выбор, формулы)
  - **Peer-Graded Assignment:** проверка другими студентами по рубрикам
  - **Programming Assignment:** автоматическая проверка кода через graders
  - **AI-Graded Assignment:** новая функция (2024-2025) — AI-оценка текстовых работ
- Дедлайны: мягкие (можно сдвигать) или жесткие
- Попытки: неограниченные для практики, ограниченные для экзаменов
- Связи: Course, Submission, Grade

**8. Submission (Отправка задания)**
- User → Assessment
- Файлы, текст, код (в зависимости от типа задания)
- Дата/время отправки
- Статус: submitted / under review / graded
- Связи: Assessment, Grade, Peer Reviews

**9. Grade (Оценка)**
- Балл: 0-100% (или проходит/не проходит для некоторых заданий)
- Источник оценки: автомат / peer / instructor / AI
- Фидбек: комментарии проверяющего или системы
- Связи: Submission, User

**10. Certificate (Сертификат)**
- Тип: Course Certificate / Specialization Certificate / Professional Certificate
- Метаданные: имя студента, название курса, дата завершения, URL верификации
- Формат: PDF для скачивания, HTML для LinkedIn
- Условия получения: завершение всех обязательных заданий + проходной балл
- Связи: User, Course/Specialization

**11. Discussion Forum (Форум обсуждений)**
- Иерархия: Course → Forum → Thread → Post → Comment (1 уровень)
- Типы тем: вопросы студентов, общие обсуждения, объявления преподавателя
- Функции: голосование (upvote), пометка "лучший ответ" (highlighted post), поиск
- Связи: Course, User (автор), Posts

**12. Learning Path (Учебный путь) — для Coursera for Business**
- Созданная администратором последовательность курсов для роли/цели
- Пререквизиты и зависимости между курсами
- Дедлайны и обязательность
- Связи: Courses, Enrollments, Skills

**13. Skill (Навык) — для Coursera for Business**
- 117+ отслеживаемых навыков (Business, Technology, Data)
- Уровень владения: beginner / intermediate / advanced / expert
- Бенчмаркинг: сравнение с индустрией
- Связи: Courses, User Skill Profile

### 2.3. ER-диаграмма (текстовое описание связей)

```
User 1──* Enrollment *──1 Course
User 1──* Enrollment *──1 Specialization
Specialization 1──* Course (упорядоченные)

User 1──* Submission *──1 Assessment
Course 1──* Assessment
Submission 1──* Grade
Submission *──* Peer Review (один студент проверяет 3+ чужих работы)

User 1──* Certificate
Course 1──* Certificate
Specialization 1──* Certificate

Course 1──* Discussion Forum 1──* Thread 1──* Post 1──* Comment
User 1──* Post/Comment

User 1──* Progress Record *──1 Item (видео/чтение/квиз)
Course 1──* Module 1──* Lesson 1──* Item

[Для Coursera for Business:]
Organization 1──* Admin User
Admin User 1──* Learning Path 1──* Course
User *──* Skill (many-to-many с уровнем владения)
```

**Ключевые паттерны:**
- **Шаблон vs Экземпляр:** Course (шаблон) → Enrollment (экземпляр для конкретного пользователя)
- **Гибкие дедлайны:** Enrollment хранит персональный график, дедлайны сдвигаются автоматически
- **Peer Review как граф:** каждый Submission связан с 3+ чужими Submissions для проверки
- **Прогресс на уровне Item:** детализация до каждого видео/чтения для точного трекинга

## 3. UX/UI особенности

### 3.1. Онбординг

**Регистрация:**
- Простая форма: email + пароль или SSO (Google, Facebook, Apple)
- Для Coursera for Business: интеграция с корпоративным SSO
- Опциональный опросник: цели обучения, интересующие темы (для персонализации)

**Первый опыт:**
- Попадание на персонализированную главную страницу с рекомендованными курсами
- Интерактивный тур по интерфейсу (опционально)
- Предложение "Browse courses" или "Enroll in your first course"

**Enrollment flow:**
1. Каталог курсов → Карточка курса с описанием, программой, отзывами, инструкторами
2. Кнопка "Enroll" → Выбор: "Audit for free" (бесплатно без сертификата) или "Purchase Certificate" (платно)
3. Для Coursera Plus: "Enroll with subscription" (доступ ко всем курсам)
4. После enrollment → автоматический переход на Course Home Page

### 3.2. Навигация

**Главная страница (Dashboard):**
- **Header:** логотип Coursera, поиск, "Explore" (каталог), иконка профиля, уведомления
- **Основная зона:**
  - **"My Courses":** карточки активных курсов с прогресс-барами и кнопкой "Continue"
  - **"Recommended for you":** AI-рекомендации на основе истории обучения
  - **Specializations/Programs in progress:** если студент записан на специализацию
- **Боковое меню (опционально):** Home, My Courses, Career, Goals, Resources

**Course Home Page:**
- **Tabs (вкладки):**
  - **Syllabus/Week:** программа по неделям/модулям с прогресс-баром
  - **Grades:** журнал оценок (какие задания сданы, баллы)
  - **Notes:** личные заметки студента
  - **Discussions:** форум курса
  - **Resources:** дополнительные материалы, загрузки
- **Next Step:** выделенная карточка "Recommended next step" с кнопкой "Start"
- **Progress:** визуальный прогресс-бар (например, "Week 2 of 6, 35% complete")

**Страница урока (Lesson Page):**
- **Левая панель:** навигация по модулям/урокам/items с чекбоксами завершения
- **Центральная зона:** контент (видео, текст, квиз)
- **Правая панель (опционально):** заметки, расшифровка видео (transcript)
- **Внизу:** кнопка "Mark as complete" и "Next" → автоматический переход к следующему item

### 3.3. Прогресс-трекинг (ключевая фишка Coursera)

**На уровне Dashboard:**
- Прогресс-бар на каждой карточке курса (например, "65% complete")
- Визуальная индикация: зеленая заливка прогресс-бара
- Дата последней активности: "Last accessed 2 days ago"

**На уровне Course Home Page:**
- Два прогресс-бара:
  1. **Общий прогресс курса:** "35% complete (Week 2 of 6)"
  2. **Прогресс текущей недели:** "3 of 7 items completed"
- **Week-by-week breakdown:** список недель с иконками завершения (зеленая галочка)
- **Детализация по типам контента:**
  - Videos: 5/8 watched
  - Readings: 3/4 completed
  - Quizzes: 2/3 passed

**"Next Step" Feature:**
- Система автоматически определяет, что делать дальше:
  - Если студент остановился на середине видео → "Continue watching: Lecture 3"
  - Если не сдал квиз → "Retry Quiz 2 (requires 80% to pass)"
  - Если завершил неделю → "Start Week 3"
- Кнопка "Start" ведет напрямую к следующему элементу (минимизация кликов)
- **Эффект:** повышение завершения курсов на >10% (данные Coursera)

**Planning Tools:**
- **Flexible Deadlines:** студент может сдвинуть дедлайны на неделю/месяц, если не успевает
- **Weekly Goals:** напоминание "Complete 3 hours this week to stay on track"
- **Notifications:** email/push о приближающихся дедлайнах, новых материалах

### 3.4. Визуальный дизайн

**Цветовая схема:**
- Основной цвет: синий (#0056D2 — фирменный Coursera blue)
- Акценты: зеленый для прогресса/завершения, красный для дедлайнов/ошибок
- Нейтральные: белый фон, светло-серые разделители, темно-серый текст

**Типографика:**
- Шрифт: Source Sans Pro (читабельный, современный sans-serif)
- Иерархия: крупные заголовки курсов, средние подзаголовки, мелкий текст для деталей

**UI-компоненты:**
- **Карточки курсов:** прямоугольные с обложкой, названием, университетом, прогресс-баром, кнопкой "Continue"
- **Прогресс-бары:** горизонтальные, с процентами и заливкой
- **Кнопки:** закругленные (rounded corners), яркие для CTA (Call to Action)
- **Иконки:** минималистичные, линейные (для видео, квизов, заданий)

**Общая эстетика:** Чистый, минималистичный, профессиональный дизайн. Фокус на контенте, а не на декоративных элементах.

### 3.5. Мобильная адаптация

**Responsive web:**
- Адаптивная верстка для планшетов и смартфонов
- Упрощенное меню (гамбургер-меню)
- Вертикальная прокрутка вместо сложных layout'ов

**Нативные приложения (iOS/Android):**
- **Offline mode:** загрузка видео и readings для просмотра без интернета
  - Квизы, пройденные оффлайн, отправляются при следующем подключении
  - Агрегированный список загрузок на Course Dashboard
- **Mobile-only features:**
  - **Notes:** скриншоты, фрагменты транскриптов, свои заметки (iOS)
  - **Reminders:** персонализированные push-уведомления ("Reminder: study at 5pm Sunday")
  - **Audio-only mode:** прослушивание лекций как подкасты
- **Синхронизация:** прогресс синхронизируется между desktop и mobile

### 3.6. Уникальные UX-фишки

1. **"Continue" вместо "Go to Course":**
   Кнопка на Dashboard всегда ведет к следующему шагу (не на главную курса), что снижает friction.

2. **Highlighted Posts в форумах:**
   Студент, задавший вопрос, может пометить лучший ответ → ответ поднимается вверх с баннером.

3. **Peer Review с AI-подстраховкой:**
   Если peer-рецензенты не проверяют задание вовремя, AI-grading может дать предварительную оценку.

4. **Skills Dashboard (для Business):**
   Визуализация навыков команды, бенчмаркинг с индустрией (170 млн учащихся).

5. **Персонализированные учебные пути (AI):**
   AI-ассистированный Course Builder для создания кастомных курсов под роли компании.

6. **Transcripts с таймкодами:**
   Клик на фразу в расшифровке → видео перематывается на этот момент.

7. **Financial Aid:**
   Встроенный процесс подачи заявки на финансовую помощь для тех, кто не может оплатить сертификат.

## 4. Ключевые процессы

### 4.1. Регистрация и запись на курс

**Сценарий: Новый пользователь регистрируется и записывается на первый курс**

```
[Студент] → Открывает Coursera.org
    ↓
[Система] → Предлагает "Explore courses" или поиск
    ↓
[Студент] → Находит курс, нажимает "View course"
    ↓
[Система] → Показывает Course Overview Page:
    - Описание, программа (syllabus), отзывы, FAQ
    - Инструкторы, партнерский университет
    - Расписание: "Starts Dec 15" или "Self-paced"
    ↓
[Студент] → Нажимает "Enroll"
    ↓
[Система] → Проверяет: зарегистрирован ли пользователь?
    - Нет → редирект на регистрацию (email/password или SSO)
    - Да → переход к выбору плана
    ↓
[Система] → Показывает варианты:
    - "Audit course for free" (без сертификата, доступ к большинству материалов)
    - "Purchase Certificate ($49-$99)" (полный доступ + сертификат)
    - "Enroll with Coursera Plus subscription" (если есть подписка)
    ↓
[Студент] → Выбирает вариант
    ↓
[Система] → Если платно → редирект на оплату (Stripe/PayPal)
              Если бесплатно/подписка → instant enrollment
    ↓
[Система] → Создает Enrollment record (User → Course)
    ↓
[Система] → Отправляет welcome email с ссылкой на Course Home
    ↓
[Студент] → Попадает на Course Home Page, видит "Week 1: Getting Started"
    ↓
[Студент] → Нажимает "Start Week 1" → начинает первое видео
```

**Особенности:**
- **Гибкие дедлайны:** при enrollment студент выбирает, хочет ли следовать рекомендованному графику или учиться в своем темпе
- **Моментальный доступ:** после enrollment студент сразу видит материалы (no approval needed)
- **Freemium модель:** возможность аудита бесплатно снижает барьер входа

### 4.2. Прохождение курса (student journey)

**Сценарий: Студент проходит типичную неделю курса**

```
[Студент] → Логинится → видит Dashboard с карточкой курса "Week 2: 40% complete"
    ↓
[Студент] → Нажимает "Continue" на карточке
    ↓
[Система] → Открывает "Next Step": Video Lecture 2.3 (где студент остановился)
    ↓
[Студент] → Смотрит видео (10 мин):
    - Может делать паузу, менять скорость (0.75x - 2x)
    - Видит transcript справа, может кликать на фразы для перемотки
    - Делает заметку (screenshot + текст) в мобильном приложении
    ↓
[Система] → Автоматически сохраняет прогресс просмотра (watched 100%)
    ↓
[Система] → Помечает video как "completed" (зеленая галочка в навигации)
    ↓
[Студент] → Нажимает "Next" → открывается Reading (PDF/HTML)
    ↓
[Студент] → Читает материал, нажимает "Mark as complete"
    ↓
[Система] → Помечает reading как "completed"
    ↓
[Студент] → Нажимает "Next" → открывается Practice Quiz (5 вопросов)
    ↓
[Студент] → Отвечает на вопросы, нажимает "Submit"
    ↓
[Система] → Мгновенная проверка → показывает результат: "4/5 correct (80%)"
    ↓
[Система] → Показывает правильные ответы и объяснения
    ↓
[Студент] → Нажимает "Next" → открывается Graded Quiz (обязательный для прохождения)
    ↓
[Студент] → Выполняет Graded Quiz (10 вопросов, 80% для прохождения)
    ↓
[Система] → Мгновенная проверка → результат: "9/10 (90%) - Passed!"
    ↓
[Система] → Обновляет Grades page, прогресс недели: "5/7 items completed"
    ↓
[Студент] → Видит: "Next: Peer-Graded Assignment (due in 5 days)"
    ↓
[Студент] → Открывает задание, читает инструкции и рубрику оценки
    ↓
[Студент] → Загружает свою работу (PDF или текст), нажимает "Submit"
    ↓
[Система] → Создает Submission, переводит студента на "Review peers' work"
    ↓
[Студент] → Проверяет 3 работы других студентов по рубрике (обязательно для получения оценки)
    ↓
[Система] → После проверки 3 работ → отправляет работу студента на peer review
    ↓
[Система] → Через 1-3 дня: студент получает 3 оценки от peers → финальная оценка (среднее)
              (или AI-grading, если peers медлят)
    ↓
[Студент] → Видит оценку и фидбек на Grades page
    ↓
[Система] → Обновляет прогресс: "Week 2: 100% complete"
    ↓
[Система] → На Dashboard: "Next Step: Start Week 3"
```

**Особенности:**
- **Seamless flow:** кнопка "Next" минимизирует клики и friction
- **Instant feedback:** для auto-graded quizzes студент сразу видит результат
- **Flexible deadlines:** если студент опаздывает, может сдвинуть дедлайны на неделю
- **Peer review loop:** студент не получит оценку, пока не проверит чужие работы (взаимность)

### 4.3. Взаимодействие с преподавателем и сообществом

**Форумы:**

```
[Студент] → Не понимает концепцию из лекции → переходит на вкладку "Discussions"
    ↓
[Система] → Показывает форум курса:
    - Секции: Announcements, General Discussion, Week 1, Week 2, etc.

[Студент] → Ищет похожий вопрос → не находит
    ↓
[Студент] → Создает новый thread: "Question about gradient descent in Week 2"

[Система] → Публикует вопрос → уведомляет Community TAs и инструкторов

[Другие студенты] → Отвечают на вопрос (1 уровень комментариев)
    ↓
[Студенты] → Голосуют за полезные ответы (upvote)
    ↓
[Студент-автор] → Возвращается к thread, находит лучший ответ

[Студент-автор] → Нажимает "Highlight this post" → ответ помечается баннером, поднимается вверх
    ↓
```

**Direct messages:**
- Студенты могут писать друг другу в private messages
- НЕТ прямых сообщений инструкторам (масштабируемость)

**Office Hours / Live Sessions (редко):**
- Некоторые курсы проводят вебинары через Zoom/Teams (не встроено в платформу)

### 4.4. Завершение курса и сертификация

**Сценарий: Студент завершает курс и получает сертификат**

```
[Студент] → Завершает все обязательные items и assessments
    ↓
[Система] → Проверяет условия сертификации:
    1. Все graded assignments сданы
    2. Итоговый балл ≥ passing grade (обычно 80%)
    3. Оплачен сертификат (или активна подписка Coursera Plus)
    ↓
[Система] → Условия выполнены → автоматически генерирует сертификат:
    - Имя студента
    - Название курса
    - Университет-партнер
    - Дата завершения
    - Уникальный URL для верификации (например, coursera.org/verify/ABC123)
    ↓
[Система] → Отправляет email: "Congratulations! Your certificate is ready"
    ↓
[Студент] → Переходит на Accomplishments page в профиле
    ↓
[Система] → Показывает сертификат:
    - PDF для скачивания
    - Кнопка "Add to LinkedIn" (автозаполнение профиля LinkedIn)
    - Кнопка "Share" (соцсети)
    ↓
[Студент] → Скачивает PDF, добавляет в LinkedIn
    ↓
[Система] → Обновляет профиль студента: Course Certificate добавлен в список достижений
    ↓
[Система] → Рекомендует следующие курсы: "Continue learning with related courses"
```

**Для Specializations:**

```
[Студент] → Завершает Course 1 of 5 в Specialization
    ↓
[Система] → Выдает Course Certificate, но НЕ Specialization Certificate
    ↓
[Система] → Показывает прогресс Specialization: "1/5 courses completed"
    ↓
[Студент] → Проходит Courses 2, 3, 4
    ↓
[Студент] → Переходит к Course 5: Capstone Project
    ↓
[Студент] → Выполняет финальный проект (часто peer-graded или instructor-graded)
    ↓
[Система] → Студент получает оценку за Capstone
    ↓
[Система] → Проверяет: все 5 курсов завершены + Capstone passed
    ↓
[Система] → Генерирует Specialization Certificate (помимо 5 Course Certificates)
    ↓
[Система] → Email + notification: "You earned a Specialization Certificate!"
```

**Особенности:**
- **Автоматическая выдача:** не требует ручного одобрения
- **Вечная валидность:** сертификаты не истекают
- **Верификация:** работодатели могут проверить подлинность по URL
- **LinkedIn integration:** один клик для добавления в профиль

## 5. Что можно взять для B3 LMS

### 5.1. Модель данных (высокий приоритет)

✅ **Паттерн "Шаблон → Экземпляр":**
- **Coursera:** Course (шаблон) → Enrollment (экземпляр для User)
- **B3 LMS:** Шаблон курса → Запись на курс (Enrollment)
- **Реализация:** Data Model Constructor B3 легко поддерживает такую структуру

✅ **Гибкая структура контента (Module → Lesson → Item):**
- **Coursera:** Course → Module (Week) → Lesson → Item (Video/Reading/Quiz)
- **B3 LMS:** Шаблон курса → Модуль → Урок → Материал (тип: видео/документ/тест)
- **Реализация:** Иерархические связи через Data Model, рекурсивная навигация

✅ **Прогресс на уровне Item:**
- **Coursera:** User Progress Record для каждого Item (video watched 100%, reading completed)
- **B3 LMS:** Сущность "Прогресс материала" (User × Item → status: not_started/in_progress/completed, progress_percent)
- **Реализация:** Связь many-to-many (User × Материал) с атрибутами статуса

✅ **Specialization как метакурс:**
- **Coursera:** Specialization 1──* Course (упорядоченные с пререквизитами)
- **B3 LMS:** Программа обучения 1──* Курс (с полем "Порядок" и "Обязательность")
- **Реализация:** Сущность "Программа обучения" с many-to-many связью к "Шаблон курса"

✅ **Certificate как отдельная сущность:**
- **Coursera:** Certificate (User × Course/Specialization, дата выдачи, URL верификации)
- **B3 LMS:** Экземпляр сертификата (User × Курс/Программа, дата, номер, PDF URL)
- **Реализация:** Шаблон сертификата → Экземпляр сертификата (из `gpt_design.md`)

✅ **Skill tracking (для корпоративного контекста):**
- **Coursera:** Skill entity (название навыка) × User (уровень владения: beginner→expert)
- **B3 LMS:** Навык × Сотрудник (уровень, дата оценки)
- **Реализация:** Справочник навыков + связь many-to-many с User через уровень

### 5.2. BPMN-процессы (высокий приоритет)

✅ **Автоматическая выдача сертификата:**

## 6. Что НЕ подходит

### 6.1. Технически сложные для B3 Platform

❌ **AI-Grading для текстовых заданий:**
- **Coursera:** AI анализирует эссе студентов и выставляет оценки по рубрикам
- **Почему НЕ:** Требует интеграции с внешними AI API (OpenAI, Google AI), обработка natural language, обучение моделей
- **Альтернатива для B3:** Peer review (упрощенная версия) или ручная проверка преподавателем в v1.0

❌ **Peer Review с полной автоматизацией:**
- **Coursera:** Сложная логика: назначение работ на проверку, контроль качества рецензий, AI-подстраховка, dispute resolution
- **Почему НЕ:**
  - Требует сложных алгоритмов распределения (граф matching)
  - Высокий риск злоупотреблений ("я тебе 100%, ты мне 100%")
  - Нужна модерация некачественных рецензий
- **Альтернатива для B3:**
  - v1.0: только instructor-graded assignments
  - v2.0: базовый peer review (без AI-подстраховки, с ручной модерацией)

❌ **Video player с advanced features:**
- **Coursera:** Собственный плеер с фичами:
  - Interactive transcripts (клик на фразу → перемотка видео)
  - Variable playback speed (0.75x - 2x)
  - In-video quizzes (пауза на определенном моменте → вопрос)
  - Note-taking с screenshot + timestamp
- **Почему НЕ:** Требует кастомной разработки видеоплеера или интеграции с продвинутыми сервисами (Kaltura, Wistia)
- **Альтернатива для B3:**
  - v1.0: встроенный HTML5 video player (базовые функции: play/pause, volume, fullscreen)
  - v2.0: интеграция с внешним видеохостингом (YouTube, Vimeo) — их плееры поддерживают speed control
  - v3.0: кастомный плеер с interactive transcripts (если бюджет позволит)

❌ **AI-ассистированный Course Builder:**
- **Coursera for Business:** AI генерирует учебные планы на основе запроса ("Create course on Salesforce for sales team")
- **Почему НЕ:** Требует AI API, обучение на большом корпусе курсов, сложная логика рекомендаций
- **Альтернатива для B3:** Ручное создание курсов через конструктор (B3 Data Model + UI Builder)

❌ **Marketplace с онлайн-оплатой:**
- **Coursera:** Встроенная платежная система (Stripe, PayPal), корзина, промокоды, налоги, возвраты
- **Почему НЕ:**
  - B3 Learning Portal ориентирован на корпоративное/государственное обучение (B2B/B2G), а не B2C
  - Не требуется монетизация в v1.0
- **Альтернатива для B3:**
  - Бесплатные курсы для внутренних сотрудников
  - Если нужна оплата (например, для внешних клиентов) → интеграция с внешним платежным шлюзом в v2.0+

### 6.2. Избыточные для корпоративного контекста B3

❌ **Gamification (badges, leaderboards):**
- **Coursera:** Баджи за достижения, таблицы лидеров, очки
- **Почему НЕ:**
  - Низкий приоритет для корпоративного обучения (сотрудники обучаются по необходимости, а не для соревнования)
  - Может создавать нездоровую конкуренцию
- **Когда можно добавить:**
  - v2.0+ как опциональная функция (если стейкхолдеры запросят)
  - Для определенных типов курсов (например, sales training — мотивация через leaderboards)

❌ **Social Learning (Communities, Groups):**
- **Coursera:** Платформенные сообщества, группы по интересам, обмен знаниями между студентами разных курсов
- **Почему НЕ:**
  - Для B3 LMS достаточно форумов внутри курса
  - Социальные функции требуют модерации, управления контентом
- **Альтернатива для B3:**
  - v1.0: форумы внутри каждого курса
  - v2.0: общий форум компании (если потребуется)

❌ **Guided Projects (облачные среды разработки):**
- **Coursera:** Интеграция с Coursera Labs — облачные IDE для практики (Python, SQL, Jupyter Notebooks прямо в браузере)
- **Почему НЕ:**
  - Требует инфраструктуры для запуска контейнеров (Docker, Kubernetes)
  - Высокая стоимость поддержки
- **Альтернатива для B3:**
  - Студенты устанавливают ПО локально (инструкции в курсе)
  - Интеграция с облачными платформами (Google Colab, Replit) через ссылки

❌ **Financial Aid Process:**
- **Coursera:** Система подачи заявок на финансовую помощь для тех, кто не может оплатить сертификат
- **Почему НЕ:** Не релевантно для корпоративного обучения (компания оплачивает обучение сотрудников)

### 6.3. Конфликтующие с ограничениями B3 Platform

❌ **Flexible Enrollment Dates (session-based courses):**
- **Coursera:** Некоторые курсы имеют фиксированные даты старта (cohort-based), другие — self-paced
- **Почему НЕ:**
  - Для B3 LMS проще поддерживать единую модель: self-paced с рекомендованными дедлайнами
  - Cohort-based требует усложнения логики (группы студентов, синхронизация дедлайнов)
- **Альтернатива для B3:**
  - v1.0: только self-paced
  - v2.0: опция "Группа обучения" (cohort) с общими дедлайнами (если потребуется для вебинаров/живых сессий)

❌ **Multi-tenancy на уровне университетов:**
- **Coursera:** Каждый университет-партнер имеет свой портал, настройки, брендинг
- **Почему НЕ:** B3 Learning Portal — единая система для одной организации (B3 Platform)
- **Когда может потребоваться:** Если B3 решит продавать Learning Portal как white-label решение другим компаниям → тогда multi-tenancy в v3.0+

## Итоги анализа

### Ключевые выводы:

1. **Coursera — эталон UX для массовых образовательных платформ.** Функция "next step" + прогресс-бары повышают завершение курсов на >10% — критически важно для B3 LMS.

2. **Модель данных Coursera (Course → Enrollment, Specialization → Certificate) напрямую применима к B3 Platform** через Data Model Constructor.

3. **BPMN-процессы Coursera (автовыдача сертификатов, напоминания о дедлайнах) легко реализуются в B3 BPMN Designer.**

4. **Для корпоративного контекста ценны функции Coursera for Business:** admin dashboard, массовое назначение курсов, learning paths, skills tracking.

5. **НЕ стоит копировать:** AI-grading, сложный peer review, marketplace — это для v2.0+ или вообще не требуется для B2B/B2G.

### Приоритеты для B3 LMS MVP:

**Must-have (v1.0):**
- Прогресс-бары на карточках курсов
- Кнопка "Продолжить" → next step
- Week-by-week структура курса
- Grades page (журнал оценок)
- Instant feedback для auto-graded quizzes
- Admin dashboard с метриками (Members / Learners / Completers)
- Автоматическая выдача сертификатов (BPMN)

**Nice-to-have (v2.0):**
- Flexible deadlines (перенос дедлайнов)
- Highlighted posts в форумах
- Learning paths с пререквизитами
- Offline mode для мобильного приложения
- Базовый peer review (без AI)

**Future (v3.0+):**
- AI-grading
- Advanced video player с interactive transcripts
- Gamification
- Marketplace
