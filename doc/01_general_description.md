# Общее описание системы B3 Learning Portal

**Версия документа:** 1.0
**Дата:** 2025-12-10
**Статус:** Утверждено

---

## 1. Назначение системы

**B3 Learning Portal** - тонкий учебный портал для обучения работе с платформой B3, реализованный средствами самой платформы B3.

Система предназначена для:

- **Организации полного цикла обучения**: от подачи заявки на курс до получения именного сертификата
- **Комфортного онбординга новых пользователей** платформы B3 с демонстрацией её возможностей на практике
- **Управления процессом обучения**: назначение курсов, выполнение заданий, контроль прогресса, взаимодействие преподаватель-студент
- **Автоматизации административных процессов**: регистрация слушателей, формирование отчётности, выдача сертификатов

### Ключевые принципы

1. **Не "ещё одна LMS"**, а тонкий комфортный портал, использующий встроенные возможности B3
2. **UX-first подход**: копирование лучших паттернов мировых образовательных платформ
3. **Встроенность**: максимальное использование native-возможностей B3 (конструкторы, BPMN, дашборды)
4. **Масштабируемость**: готовность к росту числа курсов и слушателей

## 2. Целевая аудитория

### Внешняя аудитория (обучающиеся)

**Первичная целевая аудитория**:
- Российский бизнес (корпоративный сектор)
- Государственные корпорации и ведомства
- IT-специалисты, аналитики, руководители проектов
- Сотрудники, внедряющие или использующие платформу B3

**Характеристики аудитории**:
- Разный уровень технической подготовки (от новичков до опытных пользователей)
- Высокие требования к безопасности и соответствию регуляторным требованиям
- Потребность в сертификации и документальном подтверждении квалификации
- Ограниченное время на обучение (необходим эффективный онбординг)

### Внутренняя аудитория (создатели контента)

- **Преподаватели/эксперты**: ведут курсы, проверяют задания, консультируют студентов
- **Методисты**: проектируют структуру курсов, разрабатывают задания и материалы
- **Администраторы**: управляют доступом, настраивают процессы, формируют отчётность

## 3. Ключевые цели и задачи

### Стратегические цели

1. **Демонстрация возможностей B3 "на себе"**
   - Портал сам является примером приложения на B3
   - Обучающиеся видят платформу в действии с первых шагов
   - Снижение порога входа через понятный интерфейс

2. **Обеспечение полного цикла обучения**
   - Подача заявки / покупка курса
   - Прохождение учебных материалов и заданий
   - Взаимодействие с преподавателем
   - Получение сертификата

3. **Комфортный пользовательский опыт**
   - Интуитивная навигация (паттерны Canvas, Coursera)
   - Прозрачное отслеживание прогресса
   - Минимум кликов до целевого действия
   - "Продолжить" вместо "начать сначала"

### Функциональные задачи

#### Для студентов
- Просмотр каталога курсов, регистрация/покупка
- Личный кабинет с прогрессом по всем курсам
- Выполнение заданий (загрузка файлов, ответы, практика на стендах)
- Асинхронное общение с преподавателями
- Отслеживание успеваемости и дедлайнов
- Получение и хранение сертификатов

#### Для преподавателей
- Проектирование структуры курса (шаблоны)
- Создание учебных потоков (экземпляры курсов)
- Проверка заданий с комментариями
- Проставление оценок
- Формирование табеля успеваемости
- Выдача сертификатов по правилам

#### Для администраторов
- Управление каталогом курсов
- Обработка заявок на обучение
- Настройка процессов и правил
- Формирование аналитики и отчётов
- Управление шаблонами сертификатов

## 4. Архитектурный подход

### Философия реализации

**Максимальное использование встроенных возможностей B3**:

```
┌─────────────────────────────────────────────────────┐
│  B3 LEARNING PORTAL                                 │
│                                                     │
│  Модель данных     ──► Data Model Constructor      │
│  Интерфейсы        ──► Dashboard Builder, Forms    │
│  Бизнес-процессы   ──► BPMN Designer               │
│  Отчётность        ──► Analytics & Dashboards      │
│  Сертификаты       ──► Print Form Constructor      │
│  Онбординг         ──► Onboarding Module           │
│  Интеграции        ──► RESTQL-API                  │
└─────────────────────────────────────────────────────┘
```

### Архитектурные принципы

1. **Шаблон → Экземпляр (Template → Instance pattern)**
   ```
   Шаблон курса → Экземпляр курса
   Шаблон задания → Экземпляр задания
   Шаблон сертификата → Экземпляр сертификата
   ```

2. **Процессо-ориентированность**
   - Все ключевые операции реализованы через BPMN-процессы
   - Визуальная прозрачность бизнес-логики
   - Возможность изменения без программирования

3. **Dashboard-first для студента**
   - Единая точка входа = дашборд студента
   - Прогресс-бары на каждой карточке курса
   - Кнопка "Продолжить" → следующее незавершённое задание

4. **KISS (Keep It Simple, Stupid)**
   - Простейшее решение для каждой задачи
   - Избегаем избыточной сложности
   - Предпочитаем стандартные компоненты B3

### Основные сущности данных

#### Курсы
- **Шаблон курса** - описание, структура, требования, правила сертификации
- **Экземпляр курса** - конкретный поток с датами, преподавателями, группой
- **Запись на курс (Enrollment)** - связь студента с экземпляром курса + персональные данные (креды на стенд)

#### Задания
- **Шаблон задания** - описание задачи, материалы, критерии оценки
- **Экземпляр задания** - индивидуальная копия для студента со статусом и оценкой

#### Пользователи и роли
- **Студент** - основная роль обучающегося
- **Преподаватель** - проверка заданий, консультации
- **Методист** - разработка контента
- **Администратор** - управление системой

#### Сертификаты
- **Шаблон сертификата** - макет PDF-формы
- **Экземпляр сертификата** - именной документ со ссылкой на файл

#### Коммуникации
- **Диалог по курсу** - общие вопросы
- **Диалог по заданию** - комментарии к конкретной работе

## 5. Основные возможности

### 5.1. Каталог и запись на курсы

**Паттерн**: Coursera, Udemy

- Каталог курсов с фильтрами (тип, уровень, статус)
- Карточка курса как мини-лендинг (описание, программа, преподаватель, отзывы)
- Простая регистрация (для авторизованных - 1 клик)
- Опциональная покупка/заявка с согласованием

**B3 реализация**:
- Справочник "Шаблон курса" с полями описания
- Форма "Заявка на регистрацию" с BPMN-процессом одобрения
- Dashboard с каталогом (виджет списка курсов)

### 5.2. Личный кабинет студента

**Паттерн**: Canvas Dashboard, Coursera My Courses

- Карточки "Мои курсы" с прогресс-барами
- Статусы: "Не начат", "В процессе (X%)", "Завершён"
- Кнопка **"Продолжить"** → прямо к следующему незавершённому заданию (не в корень курса!)
- Блок "Ближайшие дедлайны" (To-Do List)
- Последние сообщения от преподавателей

**B3 реализация**:
- Персональный дашборд с виджетами
- Вычисляемый показатель прогресса (count завершённых заданий / всех)
- Фильтры по статусам записей

### 5.3. Страница курса

**Паттерн**: Canvas Course Page, Brightspace Content

**Структура**:
```
┌──────────────────────────────────────────────────┐
│  Курс: "Основы B3"            [Прогресс: 45%]   │
├──────────────┬───────────────────────────────────┤
│  НАВИГАЦИЯ   │  АКТИВНЫЙ ЭЛЕМЕНТ                 │
│              │                                   │
│ ├ Модуль 1   │  Задание 3: Создание сущности    │
│   • Лекция 1 │  ─────────────────────────────    │
│   • Задание1 │  Описание задания...              │
│   • Задание2 │                                   │
│ ├ Модуль 2   │  [Загрузить файл]                │
│   • Лекция 2 │  [Отправить на проверку]          │
│   • Задание3 │                                   │
│              │  ─────────────────────────────    │
│              │  Диалог по заданию ↓              │
└──────────────┴───────────────────────────────────┘
```

**B3 реализация**:
- Форма "Запись на курс" с вкладками
- Древовидная навигация слева (компонент дерева)
- Центральная область = вложенная форма текущего задания

### 5.4. Выполнение заданий

**Паттерн**: Canvas Assignments, Brightspace Submissions

**Типы заданий**:
- Лекция (материал для изучения, чекбокс "просмотрено")
- Тест/викторина (встроенный функционал тестирования B3)
- Практическое задание (загрузка файлов, текст, ссылка на стенд)

**Жизненный цикл задания**:
```
Не начато → В работе → Отправлено → На проверке →
  → Принято / Требуется доработка → Принято
```

**B3 реализация**:
- Сущность "Экземпляр задания" со статусной моделью
- BPMN-процесс: Отправка → Создание задачи преподавателю → Проверка → Уведомление студенту
- История попыток (версионирование ответов)

### 5.5. Коммуникация

**Паттерн**: Асинхронная коммуникация (форумы, Q&A)

- Общий чат курса (вопросы по материалам)
- Чат по заданию (обсуждение конкретной работы)
- Уведомления о новых сообщениях

**B3 реализация**:
- Сущности "Диалог по курсу", "Диалог по заданию"
- Таблица "Сообщение" (child-элемент)
- Уведомления через механизм нотификаций B3

### 5.6. Отслеживание прогресса

**Паттерн**: Coursera Progress Tracking, Canvas Class Progress

**Для студента**:
- Процент завершения курса (визуальный прогресс-бар)
- Список выполненных/невыполненных заданий с галочками
- Текущие баллы и рейтинг (если применимо)

**Для преподавателя**:
- Табель "Студент × Задание" (матрица)
- Цветовая индикация статусов (зелёный = принято, жёлтый = на проверке, красный = просрочено)
- Фильтры по группам, по заданиям, по статусам

**B3 реализация**:
- Dashboard с виджетом таблицы (Gradebook)
- Вычисляемые поля прогресса
- Аналитические отчёты (модуль Аналитика B3)

### 5.7. Сертификация

**Паттерн**: Coursera Certificates, SAP Litmos Auto-Certification

**Функциональность**:
- Автоматическая генерация сертификата при выполнении условий
- Условия: завершение N из M заданий, минимальный балл
- PDF с уникальным номером, QR-кодом (опционально)
- Раздел "Мои сертификаты" в ЛК студента

**B3 реализация**:
- Конструктор печатных форм для макета сертификата
- BPMN-процесс "Завершение курса": проверка условий → создание экземпляра → генерация PDF
- Справочник "Экземпляр сертификата" с прикреплённым файлом

## 6. Референсные решения

При проектировании B3 Learning Portal использованы лучшие практики следующих платформ:

### 6.1. Canvas LMS (Instructure)
**Что взяли**:
- Чистый минималистичный UI дашборда студента
- Паттерн "одна страница курса с навигацией слева"
- Инструмент SpeedGrader (быстрая проверка заданий с комментариями)
- Календарь и To-Do список для дедлайнов

**Почему**:
Признанный лидер по UX/UI в академическом секторе. Интуитивность, доказанная миллионами пользователей.

### 6.2. Coursera
**Что взяли**:
- Прогресс-бары на каждом курсе
- Кнопка "Продолжить" → следующий шаг (не начало курса)
- Понятная структура "Модуль → Неделя → Элементы"
- Автоматические сертификаты с именами

**Почему**:
Эталон простоты онбординга. Студент всегда знает, что делать дальше. Высокий процент завершения курсов благодаря UX.

### 6.3. SAP Litmos
**Что взяли**:
- Корпоративный подход к назначению курсов (по ролям, автоматика)
- Автосертификация по правилам
- Геймификация (бейджи, таблицы лидеров)
- Брендирование портала под организацию

**Почему**:
Лидер в корпоративном B2B/B2G обучении. Простота для конечного пользователя + мощь для администратора.

### 6.4. Docebo LMS
**Что взяли**:
- Гибкий конструктор страниц (drag-and-drop виджетов)
- Персонализация интерфейса под аудитории
- Модуль e-commerce для продажи курсов
- AI-рекомендации контента (на будущее)

**Почему**:
Сочетание стильного UX и enterprise-возможностей. Опыт использования в госсекторе (FedRAMP).

### 6.5. D2L Brightspace
**Что взяли**:
- Инструмент Class Progress (детальный прогресс студента)
- Адаптивное обучение (условный релиз материалов)
- Мощная аналитика рисков (кто может не завершить курс)
- Акцент на доступности (WCAG-стандарты)

**Почему**:
Сбалансированная система для образования и корпораций. Сильная аналитика и прогнозирование.

### 6.6. Absorb LMS
**Что взяли**:
- "Consumer-grade" опыт (уровень удобства коммерческих сервисов)
- Простая навигация, понятные дашборды
- Автоматизация записи на курсы по событиям
- Поддержка нескольких порталов в одной системе

**Почему**:
Высочайшие оценки по простоте освоения. Минимальное обучение пользователей работе с системой.

### 6.7. Moodle 4.x / Totara
**Что взяли**:
- Обновлённый современный UI (Moodle 4.0 редизайн)
- Activity Completion (галочки выполнения)
- Встроенный BigBlueButton для вебинаров
- Модульность и кастомизация

**Почему**:
Самая распространённая open-source LMS в мире. Знакома российским организациям. Полный контроль над данными.

### Синтез лучших практик

| UX-паттерн | Откуда взят | Как реализуем в B3 |
|------------|-------------|---------------------|
| Dashboard с прогресс-барами | Canvas, Coursera | Dashboard Builder, виджеты |
| Кнопка "Продолжить" | Coursera | Вычисляемое поле "следующее задание" |
| Табель "студент×задание" | Brightspace, Absorb | Dashboard с таблицей и фильтрами |
| Автосертификация | Litmos, Docebo | BPMN + Print Form Constructor |
| Диалоги по заданиям | Canvas, Moodle | Сущность "Диалог" с вложенными сообщениями |
| Каталог курсов | Udemy, Coursera | Справочник с карточками |
| Activity Completion | Moodle, Brightspace | Статусная модель "Экземпляр задания" |

## 7. Инструменты платформы B3

B3 Learning Portal полностью реализуется средствами платформы B3 без необходимости в сторонних системах.

### 7.1. Интерактивный конструктор модели данных

**Назначение**: Создание всех сущностей портала (курсы, задания, студенты, диалоги, сертификаты)

**Использование в проекте**:
- Справочники: Шаблон курса, Шаблон задания, Шаблон сертификата
- Документы: Запись на курс, Экземпляр задания, Заявка на регистрацию
- Связи: One-to-Many (курс → задания), Many-to-Many (студенты ↔ курсы через Enrollment)

**Преимущества**:
- Визуальное проектирование без кода
- Автоматическая генерация CRUD-интерфейсов
- Встроенная валидация и типизация

### 7.2. BPMN Designer (визуальные процессы)

**Назначение**: Автоматизация бизнес-логики обучения

**Ключевые процессы портала**:

1. **Заявка на курс → Запись**
   ```
   Создание заявки → Согласование (опционально) →
   → Проверка оплаты → Создание Enrollment →
   → Генерация экземпляров заданий → Отправка приглашения
   ```

2. **Цикл задания**
   ```
   Студент отправил → Задача преподавателю →
   → Проверка → Оценка + комментарий →
   → Обновление статуса → Уведомление студенту
   ```

3. **Сертификация**
   ```
   Завершение всех заданий → Проверка условий (балл ≥ порога) →
   → Создание экземпляра сертификата → Генерация PDF →
   → Уведомление студенту
   ```

**Преимущества**:
- Прозрачность бизнес-логики
- Возможность изменения без программирования
- Мониторинг выполнения процессов

### 7.3. Dashboard Builder (конструктор дашбордов)

**Назначение**: Создание пользовательских интерфейсов для всех ролей

**Дашборды портала**:

- **Студент**: Мои курсы, Прогресс, Ближайшие дедлайны, Сообщения
- **Преподаватель**: Табель группы, Задания на проверке, Статистика курса
- **Администратор**: Заявки на обработке, Аналитика по курсам, Выданные сертификаты

**Типы виджетов**:
- Карточки (курсы с прогресс-барами)
- Таблицы (Gradebook)
- Графики (распределение успеваемости)
- Календари (дедлайны)
- Списки (To-Do, сообщения)

### 7.4. Конструктор интерфейсов (живые формы)

**Назначение**: Создание страниц курса, карточек заданий, диалогов

**Возможности**:
- Drag-and-drop компоновка полей
- Вкладки и секции (страница курса = Материалы, Задания, Переписка, Результаты)
- Условная видимость элементов
- Валидаторы ввода
- Загрузка файлов

**Ключевые формы портала**:
- Форма "Запись на курс" (с вложенными заданиями и диалогами)
- Форма "Экземпляр задания" (описание, решение студента, оценка, комментарии)
- Форма "Заявка на регистрацию"

### 7.5. Конструктор печатных форм

**Назначение**: Создание PDF-сертификатов

**Функциональность**:
- Визуальный редактор макета
- Поля из модели данных (ФИО, дата, номер курса)
- Логотипы, водяные знаки
- Генерация уникального номера
- Опционально: QR-код для валидации

**Шаблоны**:
- Сертификат базового курса
- Сертификат продвинутого курса
- Опционально: справка об обучении, выписка баллов

### 7.6. Модуль онбординга

**Назначение**: Создание пошаговых обучающих туров

**Использование**:
- Онбординг для нового студента (тур по интерфейсу ЛК)
- Онбординг преподавателя (как создать курс, как проверить задание)
- Внутри курса: пошаговые инструкции для заданий

**Преимущества**:
- Снижение порога входа
- Демонстрация возможностей B3 через саму систему
- Снижение нагрузки на поддержку

### 7.7. Аналитика и отчёты

**Назначение**: Формирование управленческой отчётности

**Виды отчётов**:
- Процент завершения курсов по группам
- Распределение баллов по заданиям
- Время прохождения курса (средний показатель)
- Активность студентов (логины, время в системе)
- Количество выданных сертификатов

**Визуализации**:
- Графики (столбцы, линии, круговые)
- Тепловые карты (успеваемость)
- Таймлайны (прогресс во времени)

### 7.8. RESTQL-API и интеграции

**Назначение**: Интеграция с внешними системами

**Сценарии интеграции**:

1. **HR-системы**: Автоматическая регистрация новых сотрудников на базовый курс
2. **Платёжные системы**: Обработка оплаты курсов (для коммерческих потоков)
3. **SSO**: Единый вход через корпоративный AD/LDAP
4. **Вебинарные платформы**: Zoom, MS Teams для онлайн-занятий
5. **Почтовые сервисы**: Рассылка уведомлений

**Технический стек**:
- REST API для внешних запросов
- Webhooks для событий (новая заявка, завершение курса)
- Расписания для синхронизации данных

### 7.9. Мобильный клиент

**Назначение**: Доступ к обучению с мобильных устройств

**Возможности**:
- Адаптивный веб-интерфейс
- Нативное мобильное приложение B3 (если доступно)
- Оффлайн-доступ к материалам (опционально)

## 8. Этапы внедрения (рекомендуемые)

### Фаза 1: MVP (Minimum Viable Product)
**Срок**: 1-2 месяца

**Функциональность**:
- 2 курса (базовый + продвинутый)
- Модель данных (основные сущности)
- Дашборд студента с карточками курсов
- Базовая страница курса с заданиями
- Проверка заданий преподавателем (ручная)
- Простой табель успеваемости
- Сертификат (шаблон PDF)

**Критерий успеха**: 10-20 студентов проходят пилотный курс

### Фаза 2: Автоматизация
**Срок**: 1-2 месяца

**Функциональность**:
- BPMN-процессы (заявка, задания, сертификация)
- Автоматические уведомления
- Диалоги (студент-преподаватель)
- Расширенная аналитика
- Модуль онбординга

**Критерий успеха**: Снижение ручных операций на 70%

### Фаза 3: Масштабирование
**Срок**: 2-3 месяца

**Функциональность**:
- 4-6 курсов
- Каталог курсов с фильтрами
- Заявка/покупка курса (с оплатой)
- Интеграции (HR, SSO, платежи)
- Геймификация (бейджи, рейтинги)
- Мобильная версия

**Критерий успеха**: 100+ активных студентов, 5+ курсов

## 10. Риски и ограничения

### Технические риски

1. **Ограничения B3-компонентов**
   - Некоторые UX-паттерны могут не полностью реализоваться встроенными средствами
   - **Митигация**: Приоритет B3-native, кастомизация только при необходимости

2. **Производительность**
   - При большом числе студентов возможны задержки
   - **Митигация**: Оптимизация запросов, индексы, кеширование

3. **Кривая обучения**
   - Команда должна изучить все инструменты B3
   - **Митигация**: Параллельное обучение, консультации с экспертами B3

### Организационные риски

1. **Scope Creep**
   - Желание добавить "ещё одну фичу"
   - **Митигация**: Жёсткий фокус на MVP, фазированное внедрение

2. **Недостаток контента**
   - Курсов мало, материалы требуют доработки
   - **Митигация**: Параллельная работа методистов над содержанием
